第 2 章：解線性方程組

1. 將 *A* 分解為 *L* 和 *U*，透過對左側矩陣 *A* 進行消去法。
2. 使用 *L* 和 *U* 進行前向消去，然後進行回代求解 *x* (使用 *U*)。

先前我們同時處理 *A* 和 *b*。沒有問題—只是為了精簡程式碼，我們將這兩個步驟分開。消去法的記憶體儲存在 *L* 和 *U* 中，以便我們隨時使用。使用者指南 LAPACK 評論說：「這種情況很常見，而且節省的資源很重要，但沒有提供任何關於解決單一系統的額外程式碼的資訊。」

那麼如何求解 *b* 呢？首先，將前向消去法應用到右側（乘數儲存在 *L* 中，稍後會用到）。這會將 *b* 變更為新的右側 *c*。原始系統 *Ax = b* 被分解為兩個三角系統 *Lc = b* 和 *Ux = c*。

前向和後向求解 *Lc = b*，然後求解 *Ux = c*。（5）

為了證明這是正確的，將 *Ux = c* 乘以 *L*。那麼 *LUx = Lc* 正如 *Ax = b*。
為了強調：這沒有任何新的步驟。這正是我們一直以來所做的事情。我們實際上正在求解三角系統 *Lc = b*，因為消去法向前進行。然後回代產生 *x*。一個範例說明了我們實際所做的事情。

**範例 3** 前向消去（向下）從 *Ax = b* 轉換為 *Ux = c*：

*Ax = b*  
u + 2v = 5
4u + 9v = 21
becomes
u + 2v = 5
v = 1
*Ux = c*

乘數為 4，這會將右側從 21 變更為 1。

*Lc = b* 下三角系統
[1 0] [4 1] [5] [21] = [5] [1]

*Ux = c* 上三角系統
[1 2] [0 1] [5] [1] = [3] [1]

*L* 和 *U* 可以進入原本儲存 *A* 的 *n²* 個儲存位置（現在可以忽略）。

**消去法的成本**

一個實際的問題是計算成本—或者說，計算量。我們可以在一台電腦上解決一個包含 100,000 個方程式的系統（這是一個密集的系統）。大型系統需要大約一百萬次運算。在科學計算中，成本是維度大小的立方體。我們可以輕鬆地處理一千個方程式，但處理一萬個方程式會變得困難。
