## 144 第 3 章 向量空間與子空間

### 3.3 秩與行簡化形式

數字 *m* 與 *n* 給定了線性方程組的大小，但並非其真實的大小。例如，方程 0 = 0 應當不被計算在內。如果矩陣中有兩條相同的行，則第二條會消失在消去過程中。如果行數 *r* 小於或等於 3，則第 *r* 行會變成全零，形成一個三角形矩陣 *U* 與行簡化形式 *R*。我們不希望計算零行。矩陣 *A* 的真實大小由其秩決定。

**定義** 矩陣 *A* 的秩是其樞軸列的數量。這個數字被稱為 *r*。

這個定義是計算性的，而我希望更深入地探討秩 *r*。矩陣最終會被簡化為 *r* 個非零行。從一個 3x4 的例子開始。

四個欄位
有多少樞軸？

```
A =  | 1  2  4 |
     | 1  2  5 |
     | 1  3  2 |
```

第一和第二欄 (1, 1, 1) 和 (1, 2, 3) 正在不同的方向上移動。這些將會是樞軸欄。第三欄 (2, 2, 2) 是第一欄的倍數。我們不會在第三欄看到樞軸。第四欄 (4, 5, 6) 是第一和第二欄的和。第四欄將會被沒有樞軸的欄位取代。

第四欄實際上是 3(1, 1, 1) + (1, 2, 3) 的第一和第二欄的組合。每個“自由欄位”是早期樞軸欄位的組合。它是特殊解 *s*，它告訴我們樞軸欄位的這些組合：

欄 3 = 2 (欄 1)           *s*<sub>1</sub> = (-2, 0, 1, 0)   *A*s<sub>1</sub> = 0
欄 4 = 3 (欄 1) + 1 (欄 2)   *s*<sub>2</sub> = (-3, -1, 0, 1)   *A*s<sub>2</sub> = 0

有了這些數字，我們可以得到正確的組合。系統化的方法是找到 *s*，使得 *Ax* = 0。這不會改變欄位，但它會改變組合，因為 *R* = 0 是等價於 *Ux* = 0，而 *Rx* = 0 將會從 *A* 到 *U*，然後到 *R*。

```
| 1  2  4 |   | 1  2  0 |
| 1  2  5 | → | 0  0  1 |
| 1  3  2 |   | 0  1  0 |
```

已經清楚地顯示了兩個樞軸在第一和第二欄中。矩陣 *A* 的秩是 2。繼續到 *R* = 0 會給出相同的樞軸欄。樞軸欄位的位置是 1 和 2。

*U* 有 2 個樞軸。
*R* 有 2 個樞軸。
秩(A) = 2。

“自由”欄位 3 和 4 產生零欄。它們是樞軸欄位的線性組合。例如，欄 3 是 2 倍的欄 1，欄 4 是 3 倍的欄 1 + 1 倍的欄 2。

**重要概念** 矩陣 *A* 的秩是 *A* 的欄空間的維度。
